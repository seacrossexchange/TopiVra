## 🚨 关键问题清单（必须修复）

### 1. 环境配置文件缺失 ⚠️ 高优先级

**问题描述**:
- `.env.example` 文件存在但可能不完整
- 缺少生产环境配置模板

**影响**:
- 新开发者难以快速启动项目
- 生产部署时容易遗漏关键配置

**修复方案**:
```bash
# 已存在的文件需要验证完整性
- server/.env.example
- client/.env.example

# 建议添加
- .env.production.example（生产环境模板）
```

**状态**: ✅ 已识别，需验证完整性

---

### 2. 数据库连接配置不一致 ⚠️ 高优先级

**问题描述**:
- `docker-compose.yml` 使用 MySQL 8.0
- `schema.prisma` 配置为 MySQL
- 但 `docker-compose.prod.yml` 中环境变量引用不一致

**影响**:
- 开发环境和生产环境数据库行为可能不一致
- 迁移脚本可能失败

**修复方案**:
```yaml
# 统一使用 MySQL 8.0
# 确保所有环境变量一致
DATABASE_URL=mysql://user:pass@host:3306/dbname
```

**状态**: ⚠️ 需验证

---

### 3. 端口配置混乱 ⚠️ 中优先级

**问题描述**:
- `docker-compose.yml`: server 端口 8000
- `START-DEV-WINDOWS.bat`: server 端口 3001
- `nginx.conf`: upstream backend server:3001
- 文档中混用 3000/3001/8000

**影响**:
- 开发者困惑
- 部署时端口冲突

**修复方案**:
```bash
# 建议统一标准
开发环境:
  - Frontend: 5173 (Vite 默认)
  - Backend: 8000
  - Nginx: 80/443

生产环境:
  - Frontend: 3000 (容器内)
  - Backend: 8000 (容器内)
  - Nginx: 80/443 (对外)
```

**状态**: 🔧 需要统一

---

### 4. 安全密钥默认值风险 🔒 高优先级

**问题描述**:
- `main.ts` 中检查了默认密钥，但启动脚本仍使用弱密钥
- `START-DEV-WINDOWS.bat` 硬编码了简单密钥

**影响**:
- 开发环境密钥泄露风险
- 可能被误用到生产环境

**修复方案**:
```bash
# 1. 使用 gen-keys.ps1 生成强密钥
# 2. 启动脚本应检查 .env 是否存在，而不是覆盖
# 3. 添加密钥强度验证
```

**状态**: 🔒 需加固

---

### 5. Nginx 配置问题 ⚠️ 中优先级

**问题描述**:
- `nginx.conf` 中 upstream 指向 `server:3001`
- 但 docker-compose 中 server 容器端口是 8000
- SSL 证书路径硬编码

**影响**:
- 生产环境反向代理失败
- SSL 配置不灵活

**修复方案**:
```nginx
# 修正 upstream
upstream backend {
    server server:8000;  # 与 docker-compose 一致
}

# SSL 证书使用环境变量或 volume
ssl_certificate     /etc/nginx/ssl/${SSL_CERT_NAME};
ssl_certificate_key /etc/nginx/ssl/${SSL_KEY_NAME};
```

**状态**: 🔧 需修复

---

### 6. 缺少健康检查超时配置 ⚠️ 低优先级

**问题描述**:
- Docker Compose 健康检查配置不完整
- 缺少合理的超时和重试策略

**影响**:
- 容器启动顺序可能不正确
- 服务依赖关系不稳定

**修复方案**:
```yaml
healthcheck:
  test: ["CMD", "curl", "--fail", "http://localhost:8000/health/live"]
  interval: 10s
  timeout: 5s
  retries: 5
  start_period: 60s  # 给予足够的启动时间
```

**状态**: ✅ 已配置，需验证

### 8. 生产环境 Swagger 暴露风险 🔒 中优先级

**问题描述**:

**影响**:
- API 接口信息泄露
- 安全风险

**修复方案**:
```typescript
// 强制在生产环境禁用
if (nodeEnv === 'production') {
  // 不创建 Swagger 文档
} else {
  // 仅开发环境启用
}
```

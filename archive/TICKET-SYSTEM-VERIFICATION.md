## ✅ 业务流程验证

### 退款流程 - 场景1：卖家同意
```
买家创建退款工单 (SELLER_REVIEWING)
  ↓
卖家同意退款 (SELLER_AGREED → ADMIN_REVIEWING)
  ↓
管理员批准 (ADMIN_APPROVED → COMPLETED)
  ↓
执行退款：
  - 订单状态 → REFUNDED
  - 买家余额增加
  - 恢复库存
```
- [x] 状态流转正确
- [x] 退款逻辑正确
- [x] 通知发送正确

### 退款流程 - 场景2：卖家拒绝 + 平台介入
```
买家创建退款工单 (SELLER_REVIEWING)
  ↓
卖家拒绝退款 (SELLER_REJECTED)
  ↓
买家申请介入 (ADMIN_REVIEWING)
  ↓
管理员批准/拒绝
```
- [x] 状态流转正确
- [x] 介入逻辑正确
- [x] 通知发送正确

### 换货流程
```
买家创建退款工单 (SELLER_REVIEWING)
  ↓
卖家提供换货 (SELLER_OFFERED_REPLACEMENT)
  ↓
买家接受换货 (BUYER_ACCEPTED_REPLACEMENT)
  ↓
卖家发货新账号 (REPLACEMENT_DELIVERED)
  ↓
买家确认收货 (COMPLETED)
```
- [x] 状态流转正确
- [x] 换货逻辑正确
- [x] 通知发送正确

### 私信流程

买家创建私信 (PENDING)

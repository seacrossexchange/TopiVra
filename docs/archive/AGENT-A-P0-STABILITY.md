# Agent A 执行简报（P0 稳定性专项）

标签：`A-P0-Stability`
优先级：`P0`
目标：清零“首次点击空白 / URL变化但页面不渲染 / 首次登录异常”等运营阻塞问题。

---

## 1. 执行范围
- 路由切换与页面挂载链路
- 认证初始化（auth hydration / bootstrap）
- Suspense / ErrorBoundary 行为
- SEO/head 注入机制冲突
- 首次进入关键页面（产品详情、登录、后台）

---

## 2. 必做任务

### A-1 复现矩阵与证据固化
- 首页 → 产品详情
- 首页 → 登录页
- 未登录访问受保护页
- 三角色后台首次进入

交付：每个场景的复现步骤 + 录屏 + 控制台日志

### A-2 路由与渲染链路埋点
埋点内容：
- 导航开始/完成
- 组件挂载/卸载
- 错误边界触发
- 关键异步依赖初始化顺序

交付：可定位空白问题的日志证据

### A-3 统一认证初始化策略
- 登录入口统一走 store/action
- hydration 未完成统一 loading
- token-only 状态用户信息补全

交付：首次登录与首跳稳定通过

### A-4 清理 head/SEO 冲突
- 排查并消除双机制混用
- 保留唯一 head/SEO 管理方案

交付：首页首跳不再出现卸载期异常

### A-5 压测回归
- 对关键路径执行 50 次循环回归
- 要求 0 白屏、0 未捕获异常

交付：回归记录 + 失败重现（若有）

---

## 3. 验收标准（全部满足才算完成）
1. 关键路径连续 50 次通过
2. 无“URL变了但页面空白”
3. 无“首次登录空白/异常跳转”
4. 无未解释的错误边界触发

# Agent B-F 执行简报（并行分工）

> 本文件用于并行执行分发。每个 Agent 只负责各自标签范围，避免越界改动。

## Agent B：结构收敛与历史页裁决
标签：`B-Structure-Convergence`
优先级：`P0/P1`

### 目标
三角色后台结构唯一化，消除重复页面与双轨入口。

### 任务
1. 输出页面裁决表：保留/合并/下线（含理由）
2. 路由树收口，移除重复主入口
3. 清理失效导航与死代码
4. 角色头像入口改为后台直达

### 验收
- 无重复主入口
- 导航与路由一致
- 角色边界清晰

## Agent C：卖家商品模型与上架闭环
标签：`C-Seller-ProductModel`
优先级：`P0/P1`

### 目标
让账号类与软件类商品都能有效上架、展示与交付。

### 任务
1. 商品类型：账号类/软件类/数字资源类
2. 分类体系：一级+二级
3. 国家策略：无国家/单国家/多国家（按类型控制）
4. 软件交付：zip/exe/link/key（至少一种必填）
5. 状态流：草稿→提审→发布/驳回

### 验收
- 账号商品样本1条跑通
- 软件商品样本1条跑通
- 前台可展示、可购买、可交付

## Agent D：管理员中台真功能化（广告位优先）
标签：`D-Admin-Operations`
优先级：`P0/P1`

### 目标
将“看起来有功能”升级为“可运营闭环功能”。

### 任务
1. 定义广告位模型：位置/素材/跳转/规则
2. 完成投放流程：创建→审核→上线→统计→下线
3. 统一审核状态机（商品/博客/广告）
4. 全操作留痕（审核人/时间/原因）

### 验收
- 运营可独立完成广告投放全流程
- 统计口径可解释（曝光/点击/CTR）

## Agent E：博客系统与权限阅读变现
标签：`E-Blog-Monetization`
优先级：`P0/P1`

### 目标
博客成为真实内容运营能力，不是展示页。

### 任务
1. 权限：卖家可投稿、管理员可编写、管理员审核发布
2. 状态流：草稿/待审/发布/驳回/下线
3. 阅读权限MVP：公开/登录/会员/打赏/二选一
4. 前台：摘要预览 + 解锁引导 + 解锁后全文

### 验收
- 3篇样本文章（公开/会员/打赏）全链路跑通

## Agent F：i18n 与反引流风控
标签：`F-I18n-RiskControl`
优先级：`P0/P1/P2`

### 目标
关键页面文案专业化 + 最小可用反引流能力上线。

### 任务
1. 清理五语种关键路径硬编码与技术字符
2. 统一术语表（买家/卖家/管理员）
3. 输入拦截：商品/博客/工单/消息
4. 举报与处罚记录可追溯
5. OCR/二维码检测能力预留（P2）

### 验收
- 关键页面无技术字符暴露
- 典型引流文本可拦截、可记录、可处理

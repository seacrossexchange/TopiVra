# TopiVra 48小时冲刺分配（可直接执行）

> 用途：快速推进到可运营交付临界点。优先清 P0，再推进 P1。

---

## Day 1（0-24h）

### 并行组 1（立即启动）
- Agent A：P0 稳定性专项（复现、定位、修复、压测）
- Agent B：结构裁决定版 + 路由收口
- Agent C：商品模型与上架表单核心字段落地

### Day 1 必达里程碑
1. 输出并签字《页面保留/合并/下线裁决表》
2. 路由关键P0缺陷完成首轮修复
3. 账号类/软件类上架表单核心字段可用
4. 首轮 build/test 通过（允许少量 P1 未收敛）

---

## Day 2（24-48h）

### 并行组 2（承接推进）
- Agent D：广告位与审核投放闭环
- Agent E：博客权限与审核流 + 权限阅读MVP
- Agent F：i18n关键路径清理 + 反引流拦截MVP

### Day 2 必达里程碑
1. 广告位完成“创建→审核→上线→统计”可演示
2. 博客完成“投稿→审核→发布→权限阅读”可演示
3. 关键页面技术字符清理完毕
4. 风控最小拦截链路可演示
5. 全量回归（build/test/关键e2e）

---

## 统一验收（48h 结束）

### P0 必须为 0
- 首次点击空白
- URL变化不渲染
- 登录首跳异常
- 重复主入口导致角色混乱

### P1 至少完成 80%
- 卖家商品上架闭环
- 管理后台关键运营闭环
- 博客权限阅读与审核闭环
- i18n关键路径专业化

### 交付包必须包含
1. 功能完成清单（已完成/延期）
2. 缺陷清单（P0/P1/P2）
3. 测试与回归报告
4. 运行演示录屏
